# Дизайн: именованные шаблоны заявок

**Дата:** 2026-07-10
**Проект:** P2P RUB/USDT Telegram-бот (`simple_bot.py`)
**Автор:** brainstorm с Левой

---

## Цель

Ускорить создание типовых заявок. Сейчас мерчант проходит 7 шагов
(направление → сумма → банк → платежи → курс → TTL → чаты) каждый раз, хотя
меняется по факту только **сумма**. Шаблон сохраняет все параметры кроме суммы
и чатов; при создании мерчант выбирает шаблон, вводит сумму, выбирает чаты —
готово. 7 шагов → 3.

## Решения (согласованы с пользователем)

| Вопрос | Решение |
|--------|---------|
| Сколько шаблонов на мерчанта | Несколько именованных (лимит 10) |
| Чаты в шаблоне | НЕ входят — выбираются каждый раз |
| Как создаётся шаблон | Кнопкой из готовой заявки (переиспользуем существующий флоу) |
| Переопределение параметров на лету | Нет, переменная только сумма |

---

## 1. Модель данных

Новый ключ стейта: `state["templates"]` — словарь `{str(uid): [шаблоны]}`.
Каждый шаблон:

```json
{
  "name": "СБЕР-продажа",
  "direction": "Продам RUB - возьму USDT",
  "bank": "СБЕР",
  "banks": ["СБЕР"],
  "payments": "до 2 платежей",
  "rate": "байбит средний",
  "ttl_min": 180
}
```

- Хранятся только поля, нужные для prefill сессии.
- `amount` и `target_chats` НЕ хранятся — каждый раз новые.
- Лимит: **10 шаблонов на мерчанта**.
- Шаблоны приватные: у каждого мерчанта свои, видит только он.
- `load_state`: если ключа `templates` нет — инициализировать пустым словарём
  (обратная совместимость со старым `state.json`).

---

## 2. Создание шаблона (кнопка из готовой заявки)

На сводке созданной/переопубликованной заявки добавляется кнопка
**«💾 В шаблон»** с callback `tpl:save:{bid}`.

Флоу:
1. Тык «💾 В шаблон».
2. Обработчик берёт `order = state["orders"].get(bid)`. **Snapshot полей снимается
   прямо здесь** (заявку могли бы закрыть позже). Если `order` нет —
   «❌ Заявка закрыта, шаблон не сохранить».
3. Создаётся временная сессия: `user_sessions[uid] = {"step": "template_name",
   "data": {"tpl_snapshot": {...поля из order...}}}`.
4. Бот: «💾 Введи имя шаблона (1–30 символов):».
5. Мерчант пишет имя → сохранение (см. §5 про валидацию/дубли/лимит).
6. Ответ: «✅ Шаблон "СБЕР-продажа" сохранён. Теперь /create предложит его.»

---

## 3. Использование шаблона (быстрый путь в /create)

`create_order_start` меняет первый экран:

- **Если у мерчанта есть шаблоны** — показать список:
  ```
  🏗 Создать заявку
  ┌─────────────────────┐
  │ 📋 СБЕР-продажа      │  callback tpl:use:0
  │ 📋 Альфа-покупка     │  callback tpl:use:1
  │ ➕ Новую с нуля       │  callback tpl:new
  └─────────────────────┘
  ```
  Сессия создаётся сразу: `{"step": "choose_template", "data": {...creator...}}`.
- **Если шаблонов нет** — поведение как сейчас (сразу выбор направления,
  `step="direction"`). Полная обратная совместимость.

Выбор шаблона (`tpl:use:N`):
1. Prefill `session["data"]` полями шаблона N (direction, bank, banks, payments,
   rate, ttl_min) + creator_id/username.
2. `session["step"] = "template_amount"`.
3. Бот: «💰 Введи сумму в RUB:».
4. Ввод суммы → **новый шаг `template_amount`** в `handle_message`: парсит сумму,
   кладёт в `data["amount"]`, сразу зовёт `show_chat_selection` (минуя
   банк/платежи/курс/ttl).
5. Выбор чатов (как в обычном флоу) → `finalize_order_creation`.

«➕ Новую с нуля» (`tpl:new`): `session["step"] = "direction"`, показать выбор
направления — дальше обычный флоу.

Итог: `/create` → тык шаблон → сумма → чаты → готово.

## 4. Управление шаблонами (`/templates`)

Только просмотр/удаление/переименование (НЕ создание — оно из заявки).

```
📋 Твои шаблоны:
┌──────────────────────────────┐
│ СБЕР-продажа   ✏️   🗑        │
│ Альфа-покупка  ✏️   🗑        │
└──────────────────────────────┘
```

- ✏️ (`tpl:rename:N`) → сессия `step="template_rename"`, бот просит новое имя.
- 🗑 (`tpl:del:N`) → подтверждение да/нет (`tpl:delok:N` / `tpl:delno`),
  затем удаление.
- Пустой список — «У тебя пока нет шаблонов. Создай заявку и нажми "💾 В шаблон".»
- Доступ: только `can_create_orders` (мерчанты/админы).

---

## 5. Крайние случаи и валидация

- **Дубль имени** при сохранении: «Шаблон "X" уже есть, перезаписать?»
  да (`tpl:overwrite`) / нет.
- **Лимит 10**: при попытке 11-го — «Достигнут лимит 10 шаблонов. Удали лишний
  через /templates.»
- **Имя**: 1–30 символов, обрезка пробелов; пустое — переспросить.
- **Snapshot при сохранении** снимается в момент нажатия «💾 В шаблон», а не при
  вводе имени (заявку могут закрыть в промежутке).
- **Индексы N** в callback — по позиции в списке шаблонов пользователя.
  Между показом и нажатием список меняется редко (правит только сам мерчант в
  личке), но на всякий случай — проверка границ, при выходе за диапазон
  «Шаблон не найден, открой /templates заново».

## 6. Точки интеграции в коде (найдены ревью)

| Место | Правка |
|-------|--------|
| `state`-схема | добавить `templates: {}`; `load_state` инициализирует если нет |
| `create_order_start` (1255) | если есть шаблоны — показать список вместо выбора направления |
| `on_callback` (1548) | добавить обработку префикса `tpl:` (save/use/new/rename/del/…); НЕ добавлять `tpl:` в SESSION_PREFIXES — обрабатывать отдельным блоком |
| `handle_message` (3017) | новые ветки `step`: `template_amount`, `template_name`, `template_rename` |
| `finalize_order_creation` (2773) | кнопка «💾 В шаблон» в клавиатуре |
| `handle_republish_order` (4055) | та же кнопка |
| хелпер клавиатуры | вынести `control_keyboard` в `build_order_control_keyboard(bid)`, звать в 4 местах (finalize, republish, 2× отмена edit_amount) |
| новая команда `/templates` | + регистрация хендлера в `main`, + в `set_user_commands` для merchant/admin |

### 6 подводных камней (учтены выше)

1. **Ввод суммы захардкожен вести на банк** (`handle_message:3035` ставит
   `step="bank"`). → отдельный шаг `template_amount`, не трогаем ветку `amount`.
2. **Сессия удаляется в finalize** (2845 `del user_sessions`). → snapshot из
   `order` в момент нажатия кнопки, временная сессия под ввод имени.
3. **control_keyboard скопирована 4 раза**. → хелпер.
4. **Текстовый роутер** ловит только `amount`/`custom_rate`/`editing_amount`. →
   явные ветки для template-шагов; держим на `step`, не путать с `state`.
5. **«Назад» из выбора чатов** в обычном флоу ведёт на ttl; в шаблонном ttl не
   показывался. → в шаблонном пути `back` из чатов ведёт на ввод суммы.
6. **1 чат / 0 чатов** в `show_chat_selection` — работает без изменений (не камень).

## Объём и риск

- Один файл `simple_bot.py`. Новых зависимостей нет.
- Старый путь `/create` (без шаблонов) не меняется — изоляция через новые
  `step`-значения и префикс `tpl:`.
- Риск для текущего стабильного флоу — низкий.
